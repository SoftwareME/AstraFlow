# AstraFlow

[Volver al README principal](../README.md)

**AstraFlow es un blueprint de flujo de trabajo nativo para IA que convierte ideas iniciales en software diseñado, probado y desplegable.**

Ayuda a los equipos a pasar de la planificación del producto a PRD, sistemas de diseño, direcciones visuales generadas por IA, handoff a Figma, implementación frontend, QA visual, pruebas automatizadas, correcciones iterativas y despliegue.

## Por qué usar AstraFlow

Las herramientas de IA son potentes, pero los prompts vagos suelen producir productos vagos. AstraFlow proporciona a los agentes de IA contexto estable, reglas de diseño, criterios de aceptación y puertas de calidad para trabajar de forma fiable.

## Problemas que resuelve

- Requisitos de producto poco claros
- UI generada por IA con aspecto de demo
- Separación entre diseño e ingeniería
- Pérdida de contexto por parte de los agentes de IA
- Falta de estados como loading, empty, error o disabled
- Superposición de elementos, texto cortado y mala respuesta móvil
- Falta de flujo entre idea, Figma, código, QA y despliegue

## Flujo principal

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## Inicio rápido

1. Completa `docs/00-project-context.md`.
2. Completa `docs/01-product-brief.md`.
3. Pide a tu agente de IA que lea `AGENTS.md` y la carpeta `docs/`.
4. Deja que proponga 3 direcciones de diseño.
5. Elige una dirección.
6. Continúa con sistema de diseño, implementación, QA, pruebas y preparación para despliegue.
