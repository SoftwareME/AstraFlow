# AstraFlow

[메인 README로 돌아가기](../README.md)

**AstraFlow는 막연한 아이디어를 설계되고 테스트되며 배포 가능한 소프트웨어로 전환하기 위한 AI 네이티브 프로젝트 워크플로우 블루프린트입니다.**

제품 기획, PRD, 디자인 시스템, AI 시각 방향, Figma 핸드오프, 프론트엔드 구현, 시각 QA, 자동화 테스트, 반복 수정, 배포까지 하나의 흐름으로 연결합니다.

## 왜 AstraFlow를 사용하나요

AI 도구는 강력하지만 모호한 프롬프트는 모호한 결과를 만듭니다. AstraFlow는 AI 에이전트에게 안정적인 프로젝트 맥락, 디자인 규칙, 수용 기준, 품질 게이트를 제공하여 한 줄 지시에 의존하지 않는 신뢰 가능한 작업을 가능하게 합니다.

## 해결하는 문제

- 불명확한 제품 요구사항
- 데모처럼 보이는 AI 생성 UI
- 디자인과 엔지니어링의 단절
- AI 에이전트의 컨텍스트 손실
- loading, empty, error, disabled 등의 상태 누락
- UI 겹침, 텍스트 잘림, 반응형 문제
- 아이디어에서 Figma, 코드, QA, 배포까지의 불안정한 흐름

## 핵심 워크플로우

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## 빠른 시작

1. `docs/00-project-context.md`를 작성합니다.
2. `docs/01-product-brief.md`를 작성합니다.
3. AI 에이전트에게 `AGENTS.md`와 `docs/` 폴더를 읽게 합니다.
4. 3가지 디자인 방향을 제안하게 합니다.
5. 하나를 선택합니다.
6. 디자인 시스템, 구현, QA, 테스트, 배포 준비로 진행합니다.
