# AstraFlow

[Zurück zum Haupt-README](../README.md)

**AstraFlow ist ein KI-natives Projekt-Workflow-Blueprint, das grobe Ideen in gestaltete, getestete und deploybare Software verwandelt.**

Es hilft Teams vom Produktplan über PRD, Designsystem, KI-generierte visuelle Richtungen, Figma-Handoff, Frontend-Implementierung, visuelle QA, automatisierte Tests, iterative Korrekturen und Deployment.

## Warum AstraFlow

KI-Tools sind leistungsfähig, aber unklare Prompts erzeugen oft unklare Produkte. AstraFlow gibt KI-Agenten stabilen Kontext, Designregeln, Akzeptanzkriterien und Qualitäts-Gates.

## Welche Probleme es löst

- Unklare Produktanforderungen
- KI-generierte UI, die wie eine Demo wirkt
- Drift zwischen Design und Engineering
- Kontextverlust bei KI-Agenten
- Fehlende Zustände wie loading, empty, error oder disabled
- Überlappende UI, abgeschnittener Text und schlechte Responsivität
- Kein stabiler Ablauf von Idee zu Figma, Code, QA und Deployment

## Kern-Workflow

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## Schnellstart

1. Füllen Sie `docs/00-project-context.md` aus.
2. Füllen Sie `docs/01-product-brief.md` aus.
3. Lassen Sie Ihren KI-Agenten `AGENTS.md` und den Ordner `docs/` lesen.
4. Lassen Sie 3 Designrichtungen vorschlagen.
5. Wählen Sie eine Richtung.
6. Fahren Sie mit Designsystem, Implementierung, QA, Tests und Deployment-Vorbereitung fort.
