# AstraFlow

[Back to main README](../README.md)

**AstraFlow is an AI-native project workflow blueprint for turning rough ideas into designed, tested, and deployable software.**

It helps teams move from product planning to PRDs, design systems, AI-generated visual directions, Figma handoff, frontend implementation, visual QA, automated testing, iterative fixes, and deployment.

## Why Use AstraFlow

AI tools are powerful, but vague prompts often create vague products. AstraFlow gives AI agents stable project context, design rules, acceptance criteria, and quality gates so they can work like a reliable product team instead of guessing from one message.

## What It Solves

- Unclear product requirements
- Demo-looking AI-generated UI
- Design and engineering drift
- AI agents losing project context
- Missing loading, empty, error, disabled, hover, active, and selected states
- UI overlap, clipped text, and poor responsive behavior
- Weak handoff from product idea to Figma, code, QA, and deployment

## Core Workflow

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## Quick Start

1. Fill `docs/00-project-context.md`.
2. Fill `docs/01-product-brief.md`.
3. Ask your AI agent to read `AGENTS.md` and the `docs/` folder.
4. Let it propose 3 design directions.
5. Choose one direction.
6. Continue through design system, implementation, QA, tests, and deployment readiness.
