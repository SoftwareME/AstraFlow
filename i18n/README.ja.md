# AstraFlow

[メイン README に戻る](../README.md)

**AstraFlow は、粗いアイデアを設計済み、テスト済み、デプロイ可能なソフトウェアへ変換するための AI ネイティブなプロジェクトワークフローです。**

プロダクト計画、PRD、デザインシステム、AI によるビジュアル方向性、Figma ハンドオフ、フロントエンド実装、ビジュアル QA、自動テスト、反復修正、デプロイまでを一つの流れにします。

## なぜ AstraFlow を使うのか

AI ツールは強力ですが、曖昧なプロンプトは曖昧なプロダクトを生みます。AstraFlow は AI エージェントに安定した文脈、デザインルール、受け入れ基準、品質ゲートを与え、単発の指示ではなく信頼できるプロダクトワークを可能にします。

## 解決する課題

- 要件が不明確
- AI 生成 UI がデモのように見える
- デザインと実装が分断される
- AI エージェントが文脈を失う
- loading、empty、error、disabled などの状態が不足する
- UI の重なり、文字切れ、レスポンシブ崩れ
- アイデアから Figma、コード、QA、デプロイまでの流れが不安定

## コアワークフロー

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## クイックスタート

1. `docs/00-project-context.md` を記入します。
2. `docs/01-product-brief.md` を記入します。
3. AI エージェントに `AGENTS.md` と `docs/` を読ませます。
4. 3 つのデザイン方向を提案させます。
5. 1 つを選択します。
6. デザインシステム、実装、QA、テスト、デプロイ準備へ進みます。
