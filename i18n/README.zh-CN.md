# AstraFlow

[返回主 README](../README.md)

**AstraFlow 是一个 AI 原生项目工作流蓝图，用来把粗略想法转化为完成设计、测试并可部署的软件。**

它帮助团队完成从产品规划、PRD、设计系统、AI 视觉方向、Figma 交付、前端实现、视觉 QA、自动化测试、自动修复到部署的完整流程。

## 为什么使用 AstraFlow

AI 工具很强，但模糊提示词往往只能生成模糊产品。AstraFlow 为 AI 代理提供稳定的项目上下文、设计规则、验收标准和质量门禁，让 AI 像可靠的产品团队一样工作，而不是根据一句话猜测。

## 解决什么痛点

- 产品需求不清晰
- AI 生成的 UI 很像 demo
- 设计和工程脱节
- AI 代理容易丢失项目上下文
- 缺少 loading、empty、error、disabled、hover、active、selected 等状态
- UI 元素重叠、文字裁切、响应式表现差
- 从产品想法到 Figma、代码、QA、部署之间缺少稳定流程

## 核心流程

```text
想法 -> PRD -> 设计方向 -> 设计系统 -> AI 视觉图 -> Figma 交付 -> 前端实现 -> 视觉 QA -> 测试 -> 修复 -> 部署
```

## 快速开始

1. 填写 `docs/00-project-context.md`。
2. 填写 `docs/01-product-brief.md`。
3. 让 AI 代理阅读 `AGENTS.md` 和 `docs/` 目录。
4. 让 AI 提供 3 个设计方向。
5. 选择一个方向。
6. 继续完成设计系统、实现、QA、测试和部署准备。
