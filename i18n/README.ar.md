# AstraFlow

[العودة إلى README الرئيسي](../README.md)

**AstraFlow هو مخطط عمل AI-native يحول الأفكار الأولية إلى برمجيات مصممة ومختبرة وقابلة للنشر.**

يساعد الفرق على الانتقال من تخطيط المنتج إلى PRD، وأنظمة التصميم، والاتجاهات البصرية المولدة بالذكاء الاصطناعي، وتسليم Figma، وتنفيذ الواجهة الأمامية، والفحص البصري، والاختبارات الآلية، والتحسينات المتكررة، والنشر.

## لماذا تستخدم AstraFlow

أدوات الذكاء الاصطناعي قوية، لكن التعليمات الغامضة تنتج غالبا منتجات غامضة. يمنح AstraFlow وكلاء الذكاء الاصطناعي سياقا مستقرا وقواعد تصميم ومعايير قبول وبوابات جودة.

## المشكلات التي يحلها

- متطلبات منتج غير واضحة
- واجهات مولدة بالذكاء الاصطناعي تبدو كعروض تجريبية
- انفصال بين التصميم والهندسة
- فقدان السياق لدى وكلاء الذكاء الاصطناعي
- غياب حالات مثل loading و empty و error و disabled
- تداخل العناصر، واقتطاع النص، وضعف الاستجابة
- غياب مسار ثابت من الفكرة إلى Figma والكود و QA والنشر

## سير العمل الأساسي

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## البداية السريعة

1. املأ `docs/00-project-context.md`.
2. املأ `docs/01-product-brief.md`.
3. اطلب من وكيل الذكاء الاصطناعي قراءة `AGENTS.md` ومجلد `docs/`.
4. دعه يقترح 3 اتجاهات تصميم.
5. اختر اتجاها واحدا.
6. تابع إلى نظام التصميم والتنفيذ و QA والاختبارات والاستعداد للنشر.
