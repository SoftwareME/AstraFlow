# AstraFlow

[Voltar ao README principal](../README.md)

**AstraFlow é um blueprint de fluxo de trabalho nativo para IA que transforma ideias iniciais em software desenhado, testado e implantável.**

Ele ajuda equipes a passar de planejamento de produto para PRD, sistemas de design, direções visuais geradas por IA, handoff para Figma, implementação frontend, QA visual, testes automatizados, correções iterativas e implantação.

## Por que usar AstraFlow

Ferramentas de IA são poderosas, mas prompts vagos geralmente criam produtos vagos. AstraFlow fornece aos agentes de IA contexto estável, regras de design, critérios de aceitação e gates de qualidade.

## Problemas que resolve

- Requisitos de produto pouco claros
- UI gerada por IA com aparência de demo
- Distância entre design e engenharia
- Perda de contexto por agentes de IA
- Falta de estados como loading, empty, error ou disabled
- Sobreposição de UI, texto cortado e problemas responsivos
- Falta de fluxo entre ideia, Figma, código, QA e implantação

## Fluxo principal

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## Começo rápido

1. Preencha `docs/00-project-context.md`.
2. Preencha `docs/01-product-brief.md`.
3. Peça ao seu agente de IA para ler `AGENTS.md` e a pasta `docs/`.
4. Deixe-o propor 3 direções de design.
5. Escolha uma direção.
6. Continue com sistema de design, implementação, QA, testes e preparação para implantação.
