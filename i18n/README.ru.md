# AstraFlow

[Вернуться к основному README](../README.md)

**AstraFlow — это AI-native blueprint проектного workflow, который превращает сырые идеи в спроектированное, протестированное и готовое к развертыванию ПО.**

Он помогает командам пройти путь от продуктового планирования до PRD, дизайн-системы, AI-визуальных направлений, Figma-handoff, frontend-реализации, визуального QA, автотестов, итеративных исправлений и деплоя.

## Зачем использовать AstraFlow

AI-инструменты мощные, но расплывчатые prompts часто дают расплывчатый продукт. AstraFlow дает AI-агентам стабильный контекст, правила дизайна, критерии приемки и quality gates.

## Какие проблемы решает

- Неясные продуктовые требования
- AI-generated UI, похожий на демо
- Разрыв между дизайном и инженерией
- Потеря контекста AI-агентами
- Отсутствие состояний loading, empty, error или disabled
- Наложение элементов, обрезанный текст и слабый responsive
- Нет устойчивого процесса от идеи до Figma, кода, QA и деплоя

## Основной workflow

```text
idea -> PRD -> design direction -> design system -> AI visuals -> Figma handoff -> frontend -> visual QA -> tests -> fixes -> deployment
```

## Быстрый старт

1. Заполните `docs/00-project-context.md`.
2. Заполните `docs/01-product-brief.md`.
3. Попросите AI-агента прочитать `AGENTS.md` и папку `docs/`.
4. Пусть он предложит 3 дизайн-направления.
5. Выберите одно направление.
6. Продолжайте с дизайн-системой, реализацией, QA, тестами и подготовкой к деплою.
