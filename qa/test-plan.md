# Test Plan

## Unit Tests

Use for:

- Pure functions
- Data formatting
- Validation
- State reducers

## Integration Tests

Use for:

- API workflows
- Form submission
- Authentication flows
- Data loading and error handling

## End-To-End Tests

Use for:

- Primary user workflow
- Critical purchase or conversion paths
- Admin workflows
- Regression-prone UI paths

## Accessibility Tests

Check:

- Keyboard navigation
- Focus states
- Contrast
- Labels
- Semantic headings
- Dialog behavior

## Visual Tests

Check:

- Desktop, tablet, mobile
- Loading, empty, error states
- Long text
- Dense data
- Small viewport edge cases
