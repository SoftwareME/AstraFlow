# Implementation Contract

## Product Direction

Build an MVP for Enterprise AI OS as a dense operational dashboard.

## MVP Scope

Implement these modules first:

1. Company Command Center
2. Strategy Stage Map
3. Agent Talent Cloud
4. Finance & Token Treasury
5. Project Dispatch Board

## Must-Have Data

Use realistic seed data for:

- one company
- four company stages
- five departments
- twelve agents
- twenty skills
- three active projects
- token accounts
- RMB and USD fiat accounts
- project milestone rewards
- agent performance records

## Core User Journey

1. Founder opens command center.
2. Reviews current company stage and next-stage blockers.
3. Sees missing professional capabilities.
4. HR configures agent clusters and assigns agents to roles.
5. Project owner dispatches tasks to agents.
6. Finance configures token payroll and milestone rewards.
7. System shows token budget impact and readiness score.
8. Operator reviews underperforming agent and decides reassignment or elimination.

## Design Constraints

- Use dense enterprise UI.
- Keep first screen operational.
- Do not use a landing page.
- Do not center the product around chat.
- Use tables, filters, stage maps, org trees, ledgers, and task boards.
- Include required states.

## Visual QA Requirements

- Desktop: 1440px
- Tablet: 1024px
- Mobile: 390px

No overlap, no clipped text, no unintended horizontal scroll.

## Legal/Product Disclaimer

Finance, token, payroll, tax, loan, and dividend features are product modeling tools. Real-world deployment requires legal, accounting, tax, and compliance review.
