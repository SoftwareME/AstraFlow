# Project Context

## Project Name

Enterprise AI OS

## One-Sentence Description

An AI-native company operating system that manages strategy, organizational structure, agent talent pools, project staffing, token incentives, finance, and milestone-based company growth.

## Product Thesis

Future companies can be partially operated by specialized AI agents. These agents do not need human-style meetings or social rituals; they are executable skill units that can be assigned to responsibilities, projects, workflows, and incentives.

Enterprise AI OS gives founders and operators a control plane for:

- company-level strategy
- organizational structure
- departments
- agent clusters
- role and skill binding
- project allocation
- token budgets and rewards
- real-world asset tracking
- milestone-based growth paths
- agent performance, promotion, retirement, and elimination

## Target Users

- founders building AI-native companies
- operators managing agent teams
- AI automation architects
- product and engineering leads assigning work to agents
- finance operators modeling token incentives and real-world assets
- HR/people-ops style teams managing agent capability pools

## Core Concept

Agents are skill embodiments, not simulated humans.

Examples:

- technical expert agent: implements code
- patent strategy agent: drafts patent output requirements
- tax planning agent: tracks tax obligations
- HR agent: manages agent registry, skills, allocation, retirement
- finance agent: tracks assets, token issuance, project revenue, bonus rules
- strategy agent: manages company growth path and milestone readiness

## Key Departments

### Group Strategy

Owns company growth workflow:

```text
technology startup
-> specialized and innovative enterprise
-> little giant company
-> listed company
```

For each stage, it tracks:

- required revenue
- professional capabilities needed
- hiring/agent requirements
- expected patents
- tax readiness
- possible bank loan amount
- next-stage blockers

### HR / Talent Cloud

Owns:

- agent registry
- agent clusters
- public skill library
- role binding
- organization assignment
- project assignment
- agent performance
- agent retirement/elimination

### Finance / Token Treasury

Owns:

- fiat asset tracking, such as RMB and USD
- exchange mechanisms between real-world assets and internal tokens
- scheduled token payroll
- project milestone rewards
- project dividend rules
- token budgets per agent, project, department, and company stage

### Project Operations

Owns:

- project portfolio
- task dispatch
- workflow execution
- progress nodes
- agent collaboration
- human-agent collaboration
- delivery evidence

## Non-Goals

- Do not make agents behave like human employees by default.
- Do not center the experience around chat or meetings.
- Do not claim legal, payroll, accounting, securities, tax, or investment compliance.
- Do not build a speculative crypto product as the default interpretation.
